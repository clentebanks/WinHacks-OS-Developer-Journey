taskkill /PID: cierra procesos desde CMD sin romper Windows

El comando taskkill /PID permite finalizar procesos específicos usando su identificador interno. Es útil cuando una aplicación se congela, no responde o no se puede cerrar desde el Administrador de tareas.

¿Qué hace taskkill /PID?

taskkill es una herramienta integrada de Windows para finalizar procesos desde la línea de comandos. Al usar /PID, le indicas exactamente qué proceso quieres cerrar usando su identificador.

Este método es más preciso que cerrar por nombre, porque cada proceso activo tiene un PID único durante esa sesión. Es especialmente útil después de usar comandos como netstat -ano o tasklist.

💡 Consejo WinHacks

Primero identifica el proceso con tasklist o el Administrador de tareas. Después usa taskkill /PID solo si estás seguro de qué proceso vas a cerrar.

⚠️

No cierres procesos críticos

No cierres procesos como csrss.exe, wininit.exe, services.exe, lsass.exe o procesos del sistema que no identificas. Puedes provocar cierres inesperados, errores o reinicios.

Paso a paso

1

Abre CMD como administrador

Presiona Win, escribe CMD, haz clic derecho y selecciona Ejecutar como administrador.

2

Lista los procesos activos

Usa tasklist para ver procesos y sus PID.

tasklist
3

Identifica el PID correcto

En la columna PID verás el identificador de cada proceso. No uses un PID al azar.

4

Cierra el proceso por PID

Sustituye 1234 por el PID real del proceso.

taskkill /PID 1234
5

Fuerza el cierre si no responde

Si la aplicación está congelada, puedes usar el parámetro /F.

taskkill /PID 1234 /F
6

Verifica que se cerró

Ejecuta nuevamente tasklist para confirmar que el PID ya no aparece.

tasklist

Comandos útiles

Listar procesos

tasklist

Cerrar por PID

taskkill /PID 1234

Forzar cierre

taskkill /PID 1234 /F

Cerrar por nombre de imagen

taskkill /IM notepad.exe /F

¿Cuándo usar taskkill /PID?

  • Una aplicación está congelada.
  • El Administrador de tareas no responde.
  • Quieres cerrar un proceso específico por PID.
  • Estás analizando procesos desde netstat -ano.
  • Necesitas automatizar cierres en scripts.
  • Un programa no se cierra normalmente.

✅ Resultado esperado

Windows finalizará el proceso indicado por PID. Si usas /F, el cierre será forzado y la aplicación no tendrá oportunidad de guardar cambios.

Errores comunes

Acceso denegado

Abre CMD como administrador. Algunos procesos requieren privilegios elevados.

No se encuentra el PID

El proceso pudo haberse cerrado solo o el PID cambió. Ejecuta tasklist otra vez y verifica el número correcto.

El proceso vuelve a abrirse

Puede pertenecer a un servicio, app en segundo plano o sistema de actualización. Revisa el servicio asociado antes de insistir.

Cerré algo importante

Reinicia Windows si el sistema queda inestable. Para la próxima, verifica el proceso antes de usar /F.

Preguntas frecuentes

¿taskkill /PID daña Windows?

El comando en sí no daña Windows. El riesgo está en cerrar procesos críticos o procesos que están guardando información.

¿Qué significa /F?

Significa forzar cierre. Debe usarse solo cuando el proceso no responde.

¿Funciona en Windows 11?

Sí. También funciona en Windows 10.

¿Cómo encuentro el PID de una conexión?

Usa netstat -ano. Luego busca ese PID en tasklist o en el Administrador de tareas.
