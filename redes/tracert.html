¿Qué hace este comando?

tracert muestra la ruta que siguen los paquetes desde tu computadora hasta un destino, como una página web o una dirección IP. Cada línea del resultado representa un salto entre routers o nodos de red.

Este comando es útil cuando una página carga lento, un servidor no responde bien o quieres saber en qué punto del camino se retrasa la conexión.

Si ping te dice que hay latencia alta, usa tracert para ver en qué salto empieza el retraso.

Antes de continuar

Un asterisco en tracert no siempre significa error grave. Algunos routers bloquean respuestas ICMP o no responden a pruebas de diagnóstico.

Paso a paso

1

Abre Windows Terminal o CMD

Haz clic derecho en el botón de Inicio y abre Terminal, PowerShell o Símbolo del sistema.

Captura pendiente: Windows Terminal abierto.
2

Ejecuta tracert hacia un dominio

Escribe este comando y presiona Enter:

tracert google.com

Windows mostrará la ruta que sigue tu conexión hasta llegar al destino.

Captura pendiente: resultado de tracert google.com.
3

Identifica los saltos

Cada número de la izquierda representa un salto. Normalmente el primer salto suele ser tu router local.

Captura pendiente: saltos de red resaltados.
4

Revisa los tiempos en ms

Los valores en ms indican cuánto tarda cada respuesta. Si un salto muestra tiempos muy altos, ahí puede iniciar el retraso.

Captura pendiente: tiempos en milisegundos resaltados.
5

Prueba una dirección IP

También puedes usar tracert con una IP pública:

tracert 8.8.8.8

Esto ayuda a comparar si el problema está en DNS o en la ruta de red.

Comandos básicos de tracert

Rastrear un dominio

Usa este comando para ver la ruta hacia una página web.

tracert google.com

Rastrear una IP

Usa este comando para ver la ruta hacia una dirección IP específica.

tracert 8.8.8.8

Evitar resolución de nombres

Usa -d para acelerar el resultado evitando que Windows resuelva nombres DNS.

tracert -d google.com

Limitar cantidad de saltos

Usa -h para definir el máximo de saltos que quieres analizar.

tracert -h 10 google.com

Qué debes revisar primero

Revisa dónde empiezan los tiempos altos, dónde aparecen asteriscos repetidos y si la ruta se queda detenida antes de llegar al destino.

Si el problema empieza en el primer salto, puede estar en tu red local. Si aparece después de varios saltos, puede estar en tu proveedor o en una ruta externa.

✅ Resultado esperado

Al usar tracert correctamente podrás ver por dónde viaja tu conexión, identificar saltos lentos y entender mejor si el fallo está en tu red, en tu proveedor o cerca del servidor destino.

Cuándo usar tracert

  • Cuando una página carga demasiado lento.
  • Cuando ping muestra latencia alta.
  • Cuando quieres saber dónde se corta la ruta.
  • Cuando un servidor responde desde algunas redes, pero no desde otras.
  • Cuando necesitas comparar rutas hacia diferentes destinos.
  • Cuando investigas problemas de conexión intermitente.

Errores comunes

Creer que un asterisco siempre es fallo

Algunos routers simplemente no responden a tracert. Si la ruta continúa después, probablemente no sea un problema grave.

Analizar solo el último salto

La clave está en ver desde qué salto empiezan los tiempos altos o los cortes, no solo si el destino final responde.

Confundir latencia externa con problema del PC

Si los primeros saltos están bien y el retraso aparece lejos de tu red, el problema probablemente no está en tu computadora.

Preguntas frecuentes

¿Qué significa tracert?

Significa trace route. En Windows se usa como tracert para mostrar la ruta que siguen los paquetes hacia un destino.

¿Tracert arregla Internet?

No. Tracert no repara la conexión. Sirve para diagnosticar por dónde viajan los paquetes y dónde puede empezar el retraso.

¿Qué significa un salto?

Un salto es un router o nodo intermedio por el que pasa tu conexión antes de llegar al destino.

¿Qué significan los asteriscos?

Significan que no hubo respuesta en ese intento. Puede ser por bloqueo, pérdida, firewall o porque ese router no responde a pruebas ICMP.

¿Necesito permisos de administrador?

No. Normalmente puedes usar tracert desde Terminal, CMD o PowerShell sin permisos de administrador.

¿Qué diferencia hay entre ping y tracert?

ping comprueba si un destino responde. tracert muestra la ruta y los saltos que sigue la conexión.
