¿Qué hace CMD?

CMD, también conocido como Símbolo del sistema, es una consola clásica de Windows que permite ejecutar comandos de texto para revisar información del sistema, trabajar con archivos, diagnosticar red y abrir herramientas integradas.

Aunque PowerShell es más moderno, CMD sigue siendo útil porque muchos comandos clásicos de Windows todavía se usan en soporte técnico, mantenimiento, redes y diagnóstico básico.

Antes de continuar

CMD puede ejecutar comandos sensibles. No copies comandos largos de internet si no sabes qué hacen, especialmente si eliminan archivos, modifican discos o cambian configuraciones.

Comandos seguros para empezar

Estos comandos son útiles para diagnóstico básico y normalmente no modifican configuraciones críticas del sistema.

ipconfig
ping google.com
systeminfo
tasklist
driverquery
dir

Para qué sirve cada comando

ipconfig

Muestra información de red como dirección IPv4, adaptadores y puerta de enlace. Es uno de los comandos más usados para diagnosticar conexión.

ping

Comprueba si un servidor responde. Es útil para saber si tienes conexión o si un destino está accesible.

systeminfo

Muestra información del sistema operativo, hardware, actualizaciones y configuración básica.

tasklist

Lista procesos activos desde CMD. Es útil para revisar qué se está ejecutando.

driverquery

Muestra controladores instalados en el sistema. Sirve para diagnóstico técnico básico.

dir

Lista archivos y carpetas dentro de la ubicación actual de la consola.

CMD vs PowerShell

CMD es más antiguo y está orientado a comandos clásicos de Windows. PowerShell es más moderno y permite administración avanzada, automatización y manejo de objetos.

✅ Diferencia rápida

  • CMD: ideal para comandos clásicos, red y tareas rápidas.
  • PowerShell: mejor para automatización y administración moderna.
  • Windows Terminal: aplicación para abrir CMD, PowerShell y otras consolas.

Cuándo usar CMD

  • Cuando necesitas revisar información básica de red.
  • Cuando quieres ejecutar comandos clásicos de Windows.
  • Cuando una guía técnica te pide abrir el Símbolo del sistema.
  • Cuando necesitas diagnosticar conectividad rápidamente.
  • Cuando quieres aprender fundamentos antes de pasar a PowerShell.

Errores comunes

Ejecutar comandos sin entenderlos

Algunos comandos pueden borrar archivos o cambiar configuraciones. Antes de ejecutarlos, verifica qué hacen.

Usar administrador para todo

No siempre hace falta. Para comandos de consulta, CMD normal suele ser suficiente.

Confundir CMD con PowerShell

Algunas instrucciones cambian entre CMD y PowerShell. Verifica en qué consola estás trabajando.

Preguntas frecuentes

¿CMD viene instalado en Windows?

Sí. CMD viene integrado en Windows 10 y Windows 11.

¿CMD es peligroso?

No por sí mismo. El riesgo depende del comando que ejecutes.

¿Necesito abrir CMD como administrador?

Solo cuando el comando requiera permisos elevados. Para consultas básicas no suele hacer falta.

¿CMD reemplaza a PowerShell?

No. CMD y PowerShell pueden convivir. CMD es clásico; PowerShell es más moderno y avanzado.

¿Cuál comando debería aprender primero?

Empieza con ipconfig, ping, systeminfo y tasklist.
