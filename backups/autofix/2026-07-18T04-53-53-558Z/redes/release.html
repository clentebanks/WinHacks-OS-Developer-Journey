ipconfig /release: libera la dirección IP de tu PC correctamente

El comando ipconfig /release libera la dirección IP asignada por el servidor DHCP de tu red. Es uno de los comandos más utilizados para solucionar problemas de conexión, renovar la configuración de red y restablecer la comunicación con el router.

Aprende a usar ipconfig /release

Este comando libera la dirección IP actual para que Windows pueda solicitar una nueva dirección al servidor DHCP posteriormente mediante ipconfig /renew.

¿Qué aprenderás?

  • Qué hace realmente ipconfig /release.
  • Cómo liberar correctamente la dirección IP.
  • Cuándo utilizar este comando.
  • Cómo recuperar la conexión con ipconfig /renew.
  • Cómo solucionar problemas comunes de DHCP.

¿Qué hace ipconfig /release?

Este comando solicita a Windows que abandone la dirección IP asignada por el servidor DHCP. Después de ejecutarlo, el adaptador de red pierde temporalmente su dirección IP, por lo que dejarás de tener acceso a la red hasta ejecutar ipconfig /renew o reiniciar la conexión.

Es un procedimiento completamente seguro y muy utilizado para resolver problemas relacionados con direcciones IP duplicadas, conflictos DHCP, cambios de red o fallos de conectividad.

💡 Consejo WinHacks

Normalmente ipconfig /release se utiliza junto con ipconfig /renew. Ambos comandos forman una de las soluciones más efectivas para reparar problemas de red en Windows.

Importante

Después de ejecutar este comando perderás temporalmente la conexión de red hasta obtener una nueva dirección IP mediante ipconfig /renew.

Paso a paso

1

Abre el Símbolo del sistema

Presiona Win, escribe CMD y ábrelo como administrador para evitar problemas de permisos.

2

Ejecuta el comando

Escribe el siguiente comando y presiona Enter.

ipconfig /release
3

Observa el resultado

Windows eliminará temporalmente la dirección IP del adaptador de red y perderás la conexión hasta solicitar una nueva.

4

Solicita una nueva dirección IP

Recupera la conexión ejecutando el siguiente comando.

ipconfig /renew
5

Comprueba la nueva configuración

Verifica que Windows recibió correctamente una nueva dirección IP.

ipconfig
6

Comprueba la conexión

Finalmente verifica que la red funciona correctamente utilizando el comando ping.

ping google.com

¿Cuándo utilizar ipconfig /release?

  • Problemas para obtener una dirección IP.
  • Conflictos de IP duplicada.
  • Cambio de router o red local.
  • Errores del servidor DHCP.
  • Problemas después de cambiar la configuración de red.
  • Diagnóstico de conectividad.
  • Antes de ejecutar ipconfig /renew.

✅ Resultado esperado

Windows liberará la dirección IP actual y quedará listo para solicitar una nueva al servidor DHCP mediante ipconfig /renew.

Errores comunes

No recupero Internet después de ejecutar /release

Es completamente normal. ipconfig /release elimina la dirección IP actual. Debes ejecutar ipconfig /renew para solicitar una nueva al servidor DHCP.

Media disconnected

Este mensaje indica que el adaptador de red no tiene conexión física o no está conectado a ninguna red. Comprueba el cable Ethernet o la conexión Wi-Fi antes de continuar.

No se obtiene una nueva dirección IP

Verifica que el servicio DHCP del router esté activo y ejecuta ipconfig /renew. Si el problema continúa, reinicia el router o el adaptador de red.

El comando muestra un error de acceso

Ejecuta el Símbolo del sistema como administrador e intenta nuevamente.

Preguntas frecuentes

¿ipconfig /release elimina mi conexión Wi-Fi?

No. Solo libera la dirección IP asignada. La conexión Wi-Fi permanece conectada al router.

¿Es seguro ejecutar este comando?

Sí. Es un comando oficial de Windows utilizado para solucionar problemas de red y renovar la configuración IP.

¿Funciona en Windows 11?

Sí. También funciona en Windows 10 y versiones anteriores compatibles con ipconfig.

¿Cuál es la diferencia entre /release y /renew?

/release libera la dirección IP actual. /renew solicita una nueva dirección IP al servidor DHCP. Normalmente ambos comandos se utilizan juntos.
