¿Qué hace este comando?

ipconfig es un comando de Windows que muestra información de la configuración de red del equipo. Permite ver datos como la dirección IP, la máscara de subred, la puerta de enlace predeterminada y los servidores DNS.

También puede ayudarte a resolver problemas de conexión usando opciones como /release, /renew y /flushdns.

💡 Consejo WinHacks

Si una página no carga, pero tu Internet parece funcionar, prueba primero ipconfig /flushdns. Muchas veces el problema está en la caché DNS.

Antes de continuar

Los comandos ipconfig /release y ipconfig /renew pueden desconectar temporalmente tu red mientras Windows obtiene una nueva dirección IP.

Paso a paso

1

Abre Windows Terminal o PowerShell

Haz clic derecho en el botón de Inicio y abre Terminal, PowerShell o Símbolo del sistema.

Busca tu dirección IPv4

En el resultado, localiza la línea llamada Dirección IPv4. Esa es la IP local que tu router asignó a tu equipo.

Revisa la puerta de enlace

La Puerta de enlace predeterminada normalmente es la dirección de tu router. Suele verse como 192.168.1.1 o similar.

5

Usa la información completa

Para ver más detalles de red, ejecuta:

ipconfig /all

Este comando muestra información avanzada como DNS, DHCP, dirección física y configuración completa de los adaptadores de red.

Comandos básicos de ipconfig

Ver información básica

Usa ipconfig para ver tu IP, máscara de subred y puerta de enlace.

ipconfig

Ver información completa

Usa ipconfig /all para ver todos los detalles de tus adaptadores de red.

ipconfig /all

Liberar la dirección IP

Usa ipconfig /release para liberar la IP actual asignada por el router.

ipconfig /release

Renovar la dirección IP

Usa ipconfig /renew para solicitar una nueva dirección IP al router.

ipconfig /renew

Limpiar la caché DNS

Usa ipconfig /flushdns cuando una página no carga correctamente o Windows guarda una dirección DNS antigua.

ipconfig /flushdns

Mostrar la caché DNS

Usa ipconfig /displaydns para ver entradas DNS almacenadas por Windows.

ipconfig /displaydns

Qué debes revisar primero

Cuando tengas problemas de Internet, revisa primero la Dirección IPv4, la Puerta de enlace predeterminada y los servidores DNS.

Si no aparece una puerta de enlace o tu dirección IP empieza con 169.254, Windows probablemente no recibió una IP válida del router.

✅ Resultado esperado

Al usar ipconfig correctamente podrás identificar tu IP local, comprobar si Windows está conectado al router y aplicar reparaciones básicas sin reiniciar el equipo.

Cuándo usar ipconfig

  • Cuando tu Internet falla de forma repentina.
  • Cuando necesitas saber la IP de tu computadora.
  • Cuando quieres entrar a la configuración del router.
  • Cuando una página no carga, pero otras sí.
  • Cuando cambiaste DNS y quieres limpiar la caché.
  • Cuando Windows muestra conexión limitada.

Errores comunes

No encontrar la IPv4 correcta

Si tienes varios adaptadores, como Wi-Fi, Ethernet o VPN, revisa el adaptador que realmente estás usando para conectarte a Internet.

Usar release sin renew

Si ejecutas ipconfig /release, luego debes usar ipconfig /renew para pedir una nueva IP.

Confundir IP local con IP pública

ipconfig muestra tu IP local dentro de la red. No muestra directamente la IP pública que ve Internet.

Preguntas frecuentes

¿Qué hace ipconfig?

Muestra la configuración de red de Windows, incluyendo dirección IP, puerta de enlace, máscara de subred y datos de los adaptadores.

¿Ipconfig es seguro?

Sí. Ver la configuración de red es seguro. Solo debes tener cuidado con /release y /renew, porque reinician temporalmente la asignación de IP.

¿Necesito permisos de administrador?

Para ver información básica normalmente no. Para algunas reparaciones de red, puede ser mejor abrir la terminal como administrador.

¿Qué diferencia hay entre ipconfig e ipconfig /all?

ipconfig muestra información básica. ipconfig /all muestra detalles avanzados como DNS, DHCP, dirección física y configuración completa.

¿Qué hace ipconfig /flushdns?

Limpia la caché DNS de Windows. Puede ayudar cuando una web no carga bien, cambiaste DNS o el sistema conserva una dirección antigua.

¿Qué hago si mi IP empieza con 169.254?

Esa dirección suele indicar que Windows no recibió una IP válida del router. Puedes probar reiniciar el router, revisar el cable o usar ipconfig /release y luego ipconfig /renew.
