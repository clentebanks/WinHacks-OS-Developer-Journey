¿Qué hace este comando?

ping envía pequeños paquetes de prueba a una dirección IP o dominio para comprobar si el destino responde. Es uno de los comandos más usados para diagnosticar problemas de red.

Con ping puedes saber si tu PC se comunica con el router, si tienes salida a Internet o si un servidor está tardando demasiado en responder.

💡 Consejo WinHacks

Si Internet falla, primero prueba ping 192.168.1.1. Si el router responde, el problema puede estar fuera de tu red local.

⚠️

Antes de continuar

Que un sitio no responda a ping no siempre significa que esté caído. Algunos servidores bloquean respuestas ICMP por seguridad.

Paso a paso

1

Abre Windows Terminal o CMD

Haz clic derecho en el botón de Inicio y abre Terminal, PowerShell o Símbolo del sistema.

Captura pendiente: Windows Terminal abierto.
2

Prueba la conexión con tu router

Escribe este comando y presiona Enter:

ping 192.168.1.1

Si recibes respuestas, tu PC sí se comunica con el router.

Captura pendiente: resultado de ping al router.
3

Prueba conexión con Internet

Ahora prueba con una dirección externa:

ping 8.8.8.8

Si este comando responde, tu conexión hacia Internet está funcionando.

4

Prueba con un dominio

Para comprobar también la resolución DNS, usa un dominio:

ping google.com

Si ping 8.8.8.8 funciona, pero ping google.com falla, el problema puede estar en DNS.

5

Revisa la latencia y pérdida

Observa el tiempo en ms y el porcentaje de pérdida. Una pérdida de paquetes alta puede causar cortes, lag o navegación inestable.

Captura pendiente: latencia y pérdida de paquetes resaltadas.

Comandos básicos de ping

Probar el router

Usa este comando para revisar si tu equipo se comunica con la puerta de enlace.

ping 192.168.1.1

Probar Internet por IP

Usa este comando para comprobar salida a Internet sin depender del DNS.

ping 8.8.8.8

Probar Internet por dominio

Usa este comando para comprobar conexión y resolución DNS.

ping google.com

Ping continuo

Usa -t para mantener el ping activo hasta detenerlo con Ctrl + C.

ping google.com -t

Definir cantidad de pruebas

Usa -n para elegir cuántos paquetes enviar.

ping google.com -n 10

Qué debes revisar primero

Revisa si aparecen respuestas, el tiempo en ms y la pérdida de paquetes. Estos datos indican si la conexión es estable o si hay cortes.

Si tu router responde, pero una IP externa no responde, el problema puede estar en el proveedor de Internet. Si la IP responde, pero el dominio falla, revisa DNS.

✅ Resultado esperado

Al usar ping correctamente podrás saber si tu PC se comunica con el router, si tienes salida a Internet y si la conexión presenta pérdida o latencia alta.

Cuándo usar ping

  • Cuando Internet se cae o está inestable.
  • Cuando quieres saber si el router responde.
  • Cuando una página no abre.
  • Cuando sospechas problemas de DNS.
  • Cuando tienes lag en juegos online.
  • Cuando necesitas comprobar pérdida de paquetes.

Errores comunes

Creer que todo servidor debe responder

Algunos sitios bloquean ping. Si no responde, prueba con otro dominio o una IP conocida antes de concluir que el sitio está caído.

No probar primero el router

Antes de probar sitios externos, revisa la conexión local con tu puerta de enlace usando ping 192.168.1.1.

Ignorar la pérdida de paquetes

Aunque haya respuesta, una pérdida constante puede causar cortes, videollamadas inestables y problemas en juegos online.

Preguntas frecuentes

¿Qué significa ping?

Es una prueba de comunicación entre tu equipo y un destino de red. Sirve para confirmar si hay respuesta y cuánto tarda.

¿Ping arregla Internet?

No. Ping no repara la conexión directamente. Sirve para diagnosticar dónde puede estar el problema.

¿Qué significa tiempo en ms?

Es la latencia. Indica cuánto tarda el paquete en ir al destino y regresar. Menos milisegundos normalmente significa mejor respuesta.

¿Qué significa solicitud agotada?

Significa que no se recibió respuesta a tiempo. Puede ser por caída, bloqueo ICMP, pérdida de paquetes o problemas de conexión.

¿Qué significa pérdida de paquetes?

Significa que algunos paquetes enviados no regresaron. Puede indicar red inestable, mala señal Wi-Fi, saturación o fallos del proveedor.

¿Necesito permisos de administrador?

No. Para usar ping normalmente no necesitas abrir la terminal como administrador.
