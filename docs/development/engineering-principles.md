# Engineering Principles

1. Understand before rewriting.
2. Protect production.
3. Prefer small reversible changes.
4. Make boundaries explicit.
5. Test behavior before refactoring.
6. Keep generated artifacts out of Git.
7. Treat error messages as part of the user experience.
8. Document decisions that affect future work.
9. Introduce complexity only when a demonstrated problem requires it.
10. Every session produces a commit-ready artifact.
