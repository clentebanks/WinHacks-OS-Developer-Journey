# Propósito

Este documento establece el flujo oficial para crear cualquier contenido dentro del ecosistema WinHacks.

Todo contenido deberá seguir este sistema.

No se crearán artículos, Shorts, PDFs o cursos fuera de este proceso.

---

# Filosofía del contenido

Todo contenido debe cumplir tres objetivos.

## Descubrir

Mostrar una función poco conocida.

Ejemplo

* God Mode
* Historial de Confiabilidad
* arp -a

---

## Enseñar

Explicar qué hace.

Cómo funciona.

Cuándo usarlo.

---

## Resolver

Ayudar al usuario a solucionar un problema real.

# Selección de temas

Antes de crear contenido responder:

* ¿Resuelve un problema?
* ¿Existe una búsqueda real?
* ¿Aporta algo nuevo?
* ¿Se puede demostrar?
* ¿Tiene aplicación práctica?

Si la respuesta es "no" a la mayoría, el tema no se publica.

# Estándar de experiencia personal

Siempre que sea posible, el contenido debe basarse en una experiencia real.

Ejemplos.

"Pensé que me estaban robando el Wi-Fi y así lo comprobé."

"Siempre veía estas flechas hasta que descubrí cómo quitarlas."

Este enfoque aumenta la autenticidad y fortalece la conexión con la audiencia.

---

# Estándar WinHacks para YouTube Shorts

Todos los Shorts deben cumplir:

* Gancho en los primeros 2 segundos.
* Narración en primera persona cuando aporte autenticidad.
* Problema antes de la solución.
* Demostración visual.
* Cambios visuales frecuentes para mantener la atención.
* Cierre con continuidad hacia el ecosistema WinHacks (artículo, recurso o siguiente descubrimiento).
