# Propósito

Este documento registra todos los cambios importantes realizados en WinHacks.

Su objetivo es mantener un historial claro de la evolución del proyecto, permitiendo conocer cuándo se realizaron cambios, qué se modificó y por qué.

Todo cambio relevante debe registrarse aquí antes de considerarse parte oficial del proyecto.

---

# Política de Versionado

WinHacks utiliza Versionado Semántico.

Formato:

```
MAJOR.MINOR.PATCH
```

Ejemplo

```
1.0.0
```

---

## MAJOR

Cambios importantes.

Puede modificar la arquitectura.

Ejemplo

* Nueva estructura del sitio.
* Nuevo Design System.
* Nueva Academy.
* Nuevo motor del sitio.

---

## MINOR

Nuevas funcionalidades.

Ejemplo

* Nueva categoría.
* Nuevos componentes.
* Nuevos módulos.

---

## PATCH

Correcciones.

Ejemplo

* Corrección de estilos.
* Errores SEO.
* Enlaces.
* Responsive.
* Mejoras de rendimiento.

# Principios

Todo cambio debe:

✔ Tener una razón.

✔ Estar documentado.

✔ Mantener compatibilidad cuando sea posible.

✔ Respetar la arquitectura del proyecto.

✔ Actualizar la documentación relacionada.

---

# Cambios que requieren nueva versión

Debe incrementarse la versión cuando ocurra alguno de estos casos:

* Cambio en la arquitectura.
* Cambio del Design System.
* Nuevo módulo principal.
* Nueva categoría.
* Nueva plantilla oficial.
* Nuevo componente reutilizable.
* Cambios SEO estructurales.
* Cambios importantes en Academy.

---

# Checklist antes de cerrar una versión

✔ Código estable.

✔ Documentación actualizada.

✔ Roadmap actualizado.

✔ Component Library actualizada.

✔ SEO revisado.

✔ Responsive revisado.

✔ Changelog actualizado.
