# Contributing

This repository currently documents an individual learning and development journey. These rules establish professional habits from the beginning.

## Workflow

1. Create or select an issue.
2. Create a focused branch.
3. Make one understandable change.
4. Test or verify the result.
5. Update documentation when required.
6. Use a clear conventional commit.
7. Merge only reviewed and understandable work.

## Commit Convention

```text
feat: add a new capability
fix: correct a defect
docs: update documentation
refactor: improve structure without changing behavior
test: add or improve tests
chore: maintain tooling or repository configuration
learn: complete a learning lesson or laboratory
```

Examples:

```text
chore: initialize developer journey repository
docs: document legacy engine structure
learn: complete JavaScript functions laboratory
feat: add article domain model
```

## Lesson Completion Standard

A completed lesson should include, when applicable:

- Notes
- Guided laboratory
- Independent exercise
- WinHacks OS application
- Engineering log entry
- Verification or test
- Focused commit

## Pull Requests

A pull request should explain:

- What changed
- Why it changed
- How it was verified
- What was learned
- Any remaining risk
