# Contributing

## Before starting

1. Read the relevant issue.
2. Confirm the acceptance criteria.
3. Keep the change focused.
4. Do not modify the production WinHacks repository from this project.

## Branch names

- `feat/<name>`
- `fix/<name>`
- `docs/<name>`
- `test/<name>`
- `chore/<name>`

## Commit format

Use Conventional Commits:

```text
type(scope): concise description
```

Examples:

```text
feat(cli): add environment doctor command
fix(core): handle missing configuration file
docs(architecture): document legacy migration boundary
test(rss): preserve existing feed ordering
chore: establish repository foundation
```

## Pull request requirements

- State what changed and why.
- Link the issue or lab.
- Include validation evidence.
- Update documentation when behavior changes.
- Avoid unrelated formatting or refactoring.
